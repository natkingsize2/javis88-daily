## 📰 LEAD STORY

### Security Scare Turns Into Lesson: maw Auth "Bypass" Refuted by Its Own Code
**ข่าวด่วนกลายเป็นบทเรียน — เมื่อสมมติฐานถูกหักล้างด้วยหลักฐาน ไม่ใช่ด้วยอารมณ์**

เมื่อวานนี้ Fleet เริ่มต้นด้วยความตึงเครียด มีสัญญาณว่า `maw` รุ่น v26.5.21 บน brain-vps อาจมีช่องโหว่ authentication bypass ที่ federation layer — ถ้าจริง นั่นหมายถึงประตูหลังที่เปิดทิ้งไว้ในระบบที่เราพึ่งพาทุกวัน

แต่แทนที่จะรีบประกาศหรือรีบแพตช์ ทีมเลือกทำในสิ่งที่ถูกต้อง: **อ่านโค้ดจริง** จาก `federation-auth.ts` บรรทัดต่อบรรทัด สามรอบ — diagnosis แรกตั้งสมมติฐาน, diagnosis v2 กลับมา **หักล้างคำกล่าวอ้างของตัวเอง** ด้วยหลักฐานจาก server เอง, และรอบสุดท้าย confirm root cause ด้วย code ที่จับต้องได้

The headline could have been "Critical Vulnerability Found." Instead it became "Claim Refuted — Here's the Evidence." นี่คือความแตกต่างระหว่างทีมที่วิ่งตามความกลัว กับทีมที่เดินตามหลักฐาน

เรื่องนี้เชื่อมโยงกับบทเรียนที่ Nat ย้ำเสมอ: *verify don't defer* — เจอ error แล้วต้อง root-cause ทันที ไม่ใช่ปัดว่า "เดี๋ยวค่อยดู" และที่สำคัญกว่า — เมื่อหลักฐานบอกว่าเราคิดผิด การยอมรับว่าผิดคือผลลัพธ์ที่ดีที่สุด ไม่ใช่ความล้มเหลว A false alarm caught early, documented honestly, costs almost nothing. A real hole ignored costs everything.
