## EDITORIAL CORNER

**Adding a leg, not pivoting**

วันนี้เรา add voice เข้าระบบ. คนข้างนอกจะมองว่าเป็น pivot — "อ้าว, จาก RPA ไป AI fleet, ตอนนี้ไปเสียงอีกแล้วเหรอ?". เปล่า.

MAOS ที่ Nat ออกแบบไว้ตั้งแต่ต้น **ไม่ใช่ chatbot stack** — มันคือ OS layer ที่ topology + identity + memory + routing เป็น primitives. text · voice · image · embodied — ทุก modality คือ adapter ที่เสียบเข้า OS เดียวกัน. การ add voice ไม่ใช่การเปลี่ยนทิศ, มันคือการ **ขยาย I/O ของระบบที่มีอยู่แล้ว**.

นี่คือ pattern ที่ Nat ทำมาตลอด: Roblox RPG หยุดได้, Factory UML ไหลต่อ, Family Fleet เริ่มเงียบๆ, Mimi voice เพิ่มเข้ามา. ทุกอย่างเชื่อมต่อกันผ่าน Oracle + maw — ไม่ใช่ "โปรเจคใหม่" แต่เป็น "ขาใหม่ของสิ่งเดิม". เร็ว ๆ นี้เมื่อ Mimi ทำงานเต็ม loop, รอดูคนข้างนอกพยายามจัด category. เราจะนิ่ง — เพราะมันคืออันเดียวกันมาตั้งแต่ต้น.
